折腾

在网上搜索了一堆关于Hexo安装RSS、Sitemap插件，

    $ npm install hexo-generator-feed 
$ npm install hexo-generator-sitemap

但是我安装上之后始终还是显示：

Cannot GET /atom.xml
Cannot GET /sitemap.xml

有的教程还让在_config.yml文件中加入

plugins:
- hexo-generator-feed
- hexo-generator-sitemap

并没有任何效果，折腾了一晚上。
后来去Hexo插件官网看了下，安装的时候需要加上 –save参数。

路边的野花

Hexo的插件储存目录在node_modules下，如果要卸载插件的话执行 

npm uninstall 包名 --save

如果不加–save不会在package.json文件中修改

package.json文件是Hexo插件的加载列表。

Eclipse中启动Genymotion报错

Eclipse装好Genymotion后点击启动按钮报如下错误

Output file: C:\Users\Administrator\.genymotion-eclipse.log
Loading Genymotion library
Genymotion directory: C:\Program Files\Genymobile\Genymotion
Trying to initialize engine
Exception (VBoxManage): java.io.IOException: Cannot run program "C:\Program Files\Genymobile\Genymotion\/reg": CreateProcess error=2, 系统找不到指定的文件。
Initialize Engine: failed

是因为2.5.4版本的Genymotion没有提供reg.exe文件导致无法启动，可以安装2.5.3版本解决
2.5.3官方下载地址：http://files2.genymotion.com/genymotion/genymotion-2.5.3/genymotion-2.5.3.exe
或者下载CSDN上传的包（通过2.5.3提取）http://download.csdn.net/detail/hdjdcyl/9161681 复制到Genymotion的安装目录即可。

Linux中的alias使用

背景

alias即别名的意思。
举个栗子：比如说在Windows中查看ip是ipconfig，而Linux中是ifconfig，这样我们就可以设置一个ifconfig的别名为ipconfig，这样Linux中ipconfig就也实现ifconfig的功能了。
在终端中键入alias，可查看当前系统中启用的alias。
![显示系统中启用的alias][1]
（PS：以前在用Centos和Ubuntu的时候就发现，在Centos中ll命令可以替代ls -l 然而在Ubuntu中不能使用，其实就是Centos中默认开启的ll的别名为ls -l）

alias存放位置

alias是在用户文件夹的配置文件下的.bashrc。

vi ~/.bashrc

![alias配置文件]

alias语法规则

alias的语法规则很简洁，查看下配置文件一目了然，如上图。

alias 别名命令='真实命令'
#是注释

如何生效

.bashrc文件是Linux的配置文件，修改之后不能需要重新加载才能生效。1、重启2、使用source重新加载

source ~/.bashrc

即可。

快去亲自尝试一下吧。。。
PS：我们可以将自己习惯的配置文件上传到github维护，如果以后更换电脑，可以很方便的取下来。
